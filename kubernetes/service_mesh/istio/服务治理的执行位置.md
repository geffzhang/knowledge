# 服务治理规则的执行位置

Istio 自己有 VirtualService/DestinationRule/EnvoyFilter 等多种 CRD，其中包含了很多流量治理规则。
这些治理规则具体执行的位置与 CRD 并无关联，更多的是取决于在哪里执行更合适。

**大多数治理规则都是在服务发起方 Sidecar 中执行**

表格如下：

| 治理规则   | 在服务发起方执行 | 在服务提供方执行 |
| ---------- | ---------------- | ---------------- |
| 路由管理   | ●                |                  |
| 负载均衡   | ●                |                  |
| 调用链分析 | ●                |                  |
| 服务认证   | ●                | ●                |
| 遥测数据   | ●                | ●                |
| 重试       | ●                |                  |
| 重写       | ●                |                  |
| 重定向     | ●                |                  |
| 鉴权       |                  | ●                |
